Additionally, content has many aspects beyond mere categorization depending on the user audience, intended use, intended place of use, background knowledge required, etc. However, all colloquial categorizations should still be organizable within stricter organizational norms.

    This document suggests a framework for Content and Aspects which can be organized through structured tags. A structured tag is <aspect>:<value>. By example, theme:prosperous and audience:teachers. Structured tags support better metadata and allow the same value to apply to different aspects. This is an initial proposal for a subset of eventual Themes and Categories. The intent is to start with a manageable, but extensible, corpus that fits the initial needs for Hub initiatives and tools.
